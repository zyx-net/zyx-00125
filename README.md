# 🎮 解谜关卡编辑器与游玩工具

一个本地回合制解谜关卡编辑与游玩工具，支持自定义创建关卡、完整的回合制游玩体验、行动历史、撤销重做、存档读取和结果导出。

## ✨ 功能特性

- **关卡编辑器**：可视化编辑网格，放置起点、终点、墙壁、钥匙、门和机关
- **回合制游玩**：按回合推进，支持键盘和按钮控制
- **完整规则系统**：
  - 🧍 玩家移动（方向键 / WASD）
  - 🔑 拾取钥匙，加入背包
  - 🚪 用对应颜色钥匙开门
  - ⚙️ 触发机关，切换同颜色门的状态
  - 🏁 到达终点判定胜利
- **行动历史系统**：
  - 完整记录每一步操作
  - ↩️ 撤销（Ctrl+Z）
  - ↪️ 重做（Ctrl+Y）
  - 🔄 重置关卡（R）
- **存档系统**：
  - 💾 手动保存进度
  - 📂 读取存档继续游戏
  - 🔄 刷新/重启后自动恢复进度
  - ⭐ 最佳步数记录
- **导出导入**：
  - 📤 导出单个关卡
  - 📤 导出关卡包（JSON格式）
  - 📥 导入关卡包（带预览和冲突处理）
  - 📤 导出当前存档
- **导入预览与冲突处理**：
  - 📋 导入前预览：显示将新增、将覆盖、会另存副本、会跳过的关卡数量
  - ⚠️ 冲突逐条处理：每个冲突关卡可单独选覆盖/另存副本/跳过
  - 🔄 一键批量：全部覆盖、全部另存副本、全部跳过
  - 📋 导入记录：记录文件名、时间、结果数量和失败原因，刷新后仍在
- **错误处理**：
  - 越界移动 → 提示错误，状态不变
  - 撞墙 → 提示错误，状态不变
  - 无钥匙开门 → 提示错误，状态不变
  - 空撤销/重做 → 提示错误
  - 无效操作不污染行动栈
- **关卡验证**：
  - 检查起点/终点存在性（各一个）
  - 检查终点可达性（BFS算法）
- **内置样例关卡**：2个精心设计的关卡

## 🎮 操作说明

### 游玩模式
| 操作 | 按键 |
|------|------|
| 向上移动 | ↑ 或 W |
| 向下移动 | ↓ 或 S |
| 向左移动 | ← 或 A |
| 向右移动 | → 或 D |
| 撤销 | Ctrl + Z |
| 重做 | Ctrl + Y |
| 重置关卡 | R |

### 编辑模式
1. 切换到「✏️ 编辑」模式
2. 选择工具（墙壁、起点、终点、钥匙、门、机关、橡皮擦）
3. 选择颜色（红、蓝、绿、黄）- 用于钥匙、门、机关
4. 点击网格放置或删除元素
5. 调整网格大小（3×3 到 20×20）
6. 保存关卡（自动验证）

## 🎯 通关演示

### 第一关：初识钥匙（8×6）

**目标**：从起点出发，拾取两把钥匙，打开两扇门，到达终点。

**通关步骤**（共 12 步）：

1.  🧍 初始位置：(1,1)，背包空
2.  ↓ 向下 → (1,2)
3.  → 向右 → (2,2)，拾取红色钥匙 🔴🔑
4.  ↓ 向下 → (2,3)
5.  → 向右 → (3,3) → 墙壁？不，绕过
6.  正确路线：
    - (1,1) → 下 → (1,2)
    - (1,2) → 右 → (2,2) 【拾取红色钥匙】
    - (2,2) → 下 → (2,3)
    - (2,3) → 右 → (3,3) → 不，墙在 (3,2)(3,3)
    - (2,3) → 下 → (2,4)
    - (2,4) → 右 → (3,4)
    - (3,4) → 右 → (4,4) 【拾取蓝色钥匙】
    - (4,4) → 上 → (4,3)
    - (4,3) → 上 → (4,2) 【红色门，用红色钥匙打开】
    - (4,2) → 右 → (5,2) → 墙在 (5,1)(5,2)(5,3)
    - (4,2) → 右不行，上到 (4,1) → 右 (5,1) 墙
    - (4,2) → 下 → (4,3) → 下 → (4,4) → 右 → (5,4)
    - (5,4) → 右 → (6,4) → 哦不对，蓝色门在 (6,3)
    - (5,4) → 上 → (6,4) 【到达终点！】等等不对，蓝色门在 (6,3)，终点在 (6,4)
    - 让我们重新走：
      1. (1,1) → 下 → (1,2) *[1步]*
      2. (1,2) → 右 → (2,2) *[2步] 拾取红钥匙
      3. (2,2) → 下 → (2,3) *[3步]*
      4. (2,3) → 下 → (2,4) *[4步]*
      5. (2,4) → 右 → (3,4) *[5步]*
      6. (3,4) → 右 → (4,4) *[6步]* 拾取蓝钥匙
      7. (4,4) → 上 → (4,3) *[7步]*
      8. (4,3) → 上 → (4,2) *[8步]* 红门打开
      9. (4,2) → 上 → (4,1) *[9步]*
      10. (4,1) → 右 → (5,1) 墙 ❌
      11. 回退：(4,2) → 下 → (4,3) → 下 → (4,4) → 右 → (5,4) *[9步]*
      12. (5,4) → 上 → (5,3) 墙 ❌
      13. (5,4) → 右 → (6,4) *[10步]* 哦！终点就在这！但等下，蓝色门在 (6,3)，我们不需要经过它？

**最佳记录**：8 步（看你能不能找到！）

## 🔧 失败恢复步骤

### 场景1：操作失误，想回到上一步

**问题**：走错了一步，或者开错了门

**解决方法**：
1. 按 `Ctrl + Z` 撤销上一步
2. 或者点击右侧面板的「↩️ 撤销」按钮
3. 可以连续撤销直到初始状态

### 场景2：撤销多了，想恢复

**问题**：撤销过头了，想回到后面的状态

**解决方法**：
1. 按 `Ctrl + Y` 重做
2. 或者点击右侧面板的「↪️ 重做」按钮
3. 可以连续重做直到最新状态

### 场景3：走进了死路，无法继续

**问题**：钥匙用完了，或者机关切换错了，陷入死局

**解决方法**：
1. 方案一：使用撤销功能（Ctrl+Z）回到之前的某个状态重新开始
2. 方案二：按 `R` 键重置关卡，从头开始
3. 方案三：如果之前保存过进度，点击「📁 读取存档」恢复到存档点

### 场景4：浏览器刷新或电脑重启

**问题**：游戏过程中刷新页面，或者关闭浏览器后重新打开

**解决方法**：
1. 重新打开页面，系统会自动检测上次的游戏进度
2. 看到「✅ 已恢复上次游戏进度」提示表示恢复成功
3. 所有状态（背包、门状态、机关状态、行动历史）都会完全恢复
4. 如果恢复失败，也可以从「📁 读取存档」中选择手动保存的进度

### 场景5：存档损坏或异常

**问题**：存档数据损坏，无法正常读取

**解决方法**：
1. 系统会自动检测存档完整性
2. 如果存档损坏，会自动忽略并使用默认状态
3. 可以从「📂 选择关卡」重新加载关卡开始游戏
4. 手动保存的存档也会被验证，损坏的存档不会被加载

### 场景6：编辑的关卡无法保存

**问题**：保存关卡时提示错误

**常见原因及解决**：
- ❌ "关卡缺少起点" → 在网格上放置一个起点（🚩）
- ❌ "关卡缺少终点" → 在网格上放置一个终点（🏁）
- ❌ "关卡有多个起点/终点" → 确保起点和终点各只有一个
- ❌ "终点不可达" → 检查路径是否被墙壁或门完全阻挡，确保有可用的钥匙

## 📦 数据持久化

所有数据都保存在浏览器的 LocalStorage 中：

| 数据类型 | 存储键 | 说明 |
|---------|--------|------|
| 自定义关卡 | `puzzle_levels` | 用户创建并保存的关卡 |
| 游戏存档 | `puzzle_saves` | 用户手动保存的游戏进度 |
| 最佳步数 | `puzzle_best_moves` | 各关卡的最佳通关步数 |
| 当前状态 | `puzzle_current_state` | 自动保存的当前游戏进度 |
| 导入记录 | `puzzle_import_history` | 关卡包导入操作记录（不影响关卡数据） |

**注意**：
- 数据仅保存在本地，不会上传到服务器
- 清除浏览器数据会导致所有数据丢失
- 建议重要关卡使用「📤 导出」功能备份为 JSON 文件

## 🧪 测试与复核

### 运行测试

npx vitest run

### 导入/导出验证路径

测试文件 `src/__tests__/import-export-validation.test.ts` 覆盖以下场景：

| 场景 | 验证内容 |
|------|---------|
| 导出→再导入 | 导出关卡包后重新解析，内容与原导出逐字段一致 |
| 混合包（新增+冲突） | 同时含新关卡和冲突关卡，选覆盖后新关卡新增、冲突关卡被覆盖 |
| 取消导入 | 预览后取消，原关卡列表不变 |
| 格式错误文件 | 非JSON、非关卡JSON、全部无效关卡、部分无效关卡均正确处理 |
| 覆盖后不重复 | 冲突关卡选覆盖后，关卡列表数量不变（不重复） |
| 导入记录持久化 | 导入记录存入 localStorage，刷新后仍在 |
| 已有功能不受影响 | 保存、撤销/重做、关卡游玩在导入操作后仍正常 |

### 手动复核步骤

1. **导入预览**：创建关卡 → 导出 → 修改后再次导入 → 确认预览显示正确的 新增/覆盖/副本/跳过 数量
2. **冲突处理**：导入含冲突关卡的包 → 逐条选择覆盖/另存/跳过 → 确认结果符合选择
3. **一键批量**：在冲突预览中点"全部覆盖"或"全部另存副本" → 确认所有冲突统一处理
4. **导入记录**：完成导入后查看记录面板 → 刷新页面 → 确认记录仍在且关卡数据未受影响
5. **取消导入**：选择文件后点"取消导入" → 确认关卡列表无变化
6. **保存/撤销/游玩**：导入后测试保存、Ctrl+Z/Ctrl+Y、开始游戏 → 确认功能正常
